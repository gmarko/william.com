Project Overview

Friday was designed to be a web based personal assistant that could keep track of all my online data. This included passwords and notes thus far. It allowed other users to sign up and use the service as well. This project is powered by Angular 4 and Google Firebase. Keeping all the technology within Google's ecosystem allowed for easy integration between the front-end and the back-end. The hosting, DNS, Database, and Authentication are all managed in the Firebase ecosystem.

Functionality

When viewing your saved accounts.

Once a user logs in to the application, they are presented with a side navigation menu with links to view the user's notes, quicknotes, and accounts. Notes are long text documents that can only be edited when they are logged in. Quicknotes are notes that can be quickly added from the landing page without even having to login, and are added by simply providing an email address. To add/view accounts, they need to enter their own pin that is used to encrypt the account passwords. This pin is never kept on any database and is the user's responsibility to keep track of it, to make sure that the accounts are fully secured.

When creating an account, the user is able to generate a password with a variety of options. They are also able to add a note to be stored with the account.

Authentication

Authentication in Friday is handled with Firebase Authentication services. This allows the application to easily offer Google OAuth sign in functionality. This also allows token communication with the server so the user doesn't have to login every time they visit the website. The login modal also offers a forgot password link, where the user can enter their email and receive a password reset email.

Firebase Real-Time Database

All of the data for this application is hosted on Firebase's Real-Time Database. This allowed for easy access by the angular client, and enable easy CRUD and query operations on the data. The firebase database database is much less sophisticated than something like MongoDB, but is more than capable for a simple application like Friday. It's key features are that it is quick to set-up and use.

There are certain nodes of the database that are secured and cannot be accessed by every user. These nodes are only accessible by Google Cloud Functions, which offers a secure way to interact with the database, making sure that a malicious attacker cannot get access to other user's personal data. All of this is enabled by the security rules of the Firebase Database and Google Cloud Functions.

DNS

Setting up the DNS for this application was actually quite challenging. First, when I first built friday with Rails and MongoDB, the DNS was managed by CloudFlare after changing the namespace on GoDaddy. Another alternatives I used were Route53 on AWS.

Luckily, Google Cloud Platform offers DNS hosting and now manages all the DNS for assistantfriday.com. The application is hosted by firebase and the DNS namespace is also directed to firebase.
