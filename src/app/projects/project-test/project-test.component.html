ERC20 Token Exchange
Completed November 2017

Project Overview

The Token Exchange is a platform users can use to exchange various ERC20 compliant tokens for ether, using an easy to use interface to post sales and view listings. The exchange contract stores both tokens and ether for users to easily exchange between the two. The exchange smart contract is fully tested using truffle tests, and is currently deployed on the ethereum Rinkeby network at address 0xe4b389a95263743320d3665a6dbed89fb12de86d. Users can interact with the contract using a client-side ethereum node such as metamask or the mist browser.

The Token Exchange currently running on the Rinkeby ethereum network and the front-end application can be found here.

Functionality

Overview of the manage Tokens page.

When a user visits the token exchange website, they are greeted with the page where they can deposit/withdraw tokens and ether. The exchange can easily trade any ERC20 compliant token, since the current available token you see now is ERC20 compliant. The exchange handles all of the token exchange logic for the client, so that the client only needs to choose how much ether they want to buy/sell their tokens for.

When viewing the manage token page.

This is where the administrator (owner) of the exchange contract can add additional ERC20 tokens to traded on the exchange. Before a user can add tokens to the exchange, they will need to approve the exchange at the ERC20 token contract, so that the exchange can withdraw the tokens from the user's account. This is also done in this page. The user can also send tokens directly to another account here.

The Buy/Sell Token page.

In this page the user can view all available buy/sell prices for a particular token.

Authentication

Since this is a distributed application, all authentication is handled client-side by something like metamask or mist. Or geth if you are interacting directly with the exchange contract. Authentication is secured inherently by the ethereum network, which verifies all of your transactions with cryptography.

storage Storage

All of the data that is necessary for this application, including your user token/ether balance is all stored on the ethereum network, meaning every node has access to it. This means everyone will be able to see how many tokens or ether you have, but also means that all of your data is super secure behind the security guaranteed by the ethereum network.

Events

In order to keep the front-end updated and make sure everyone can see updated buy/sell orders when they are available, a wide variety of events were needed. Here are all the events emitted by the token exchange contract. 

// Events for deposit/withdrawal
event DepositForTokenReceived(address indexed _from, uint indexed _symbolIndex, uint _amount, uint _timestamp);
event WithdrawalToken(address indexed _to, uint indexed _symbolIndex, uint _amount, uint _timestamp);
event DepositForEthReceived(address indexed _from, uint _amount, uint _timestamp);
event WithdrawalEth(address indexed _to, uint _amount, uint _timestamp);

// Events for orders
event LimitSellOrderCreated(uint indexed _symbolIndex, address indexed _who, uint _amountTokens, uint _priceInWei, uint _orderKey);
event SellOrderFulfilled(uint indexed _symbolIndex, uint _amount, uint _priceInWei, uint _orderKey);
event SellOrderCancelled(uint indexed _symbolIndex, uint _priceInWei, uint _orderKey);
event LimitBuyOrderCreated(uint indexed _symbolIndex, address indexed _who, uint _amountTokens, uint _priceInWei, uint _orderKey);
event BuyOrderFulfilled(uint indexed _symbolIndex, uint _amount, uint _priceInWei, uint _orderKey);
event BuyOrderCancelled(uint indexed _symbolIndex, uint _priceInWei, uint _orderKey);

// Events for management
event TokenAddedToSystem(uint _symbolIndex, string _token, uint _timestamp);
The 11 events that are emitted by the exchange token contract.

The front-end application will listen to these events and update the buy/sell order lists appropriately. It will also listen for token/ether deposits and withdrawals, making sure the front-end balances are always updated. These events can also be filtered so that the front-end only listens to deposits/withdrawals for their own account, for example.

Testing

Because ethereum contracts cannot be altered once deployed onto a public network, they need to be thoroughly tested in development. The token exchange is very thoroughly tested using truffle, and makes sure that all of the features are working and that there are no bugs before deployment. The contract is tested using chai and mocha.
